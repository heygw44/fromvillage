# fromVillage One-pager

## 1. 프로젝트 개요

`fromVillage`는 농촌·어촌 지역 생산자와 도시 소비자를 연결하는 직거래 중개 플랫폼이다.
생산자는 직접 재배·생산한 농산물과 수산물을 등록하고, 소비자는 이를 탐색하고 장바구니 및 주문 기능을 통해 구매할 수 있다.

이 프로젝트는 단순한 커머스 CRUD 구현을 넘어서, 인증/인가/보안, 주문 및 재고 정합성, 한정 수량 쿠폰의 동시성 처리, 정산 배치를 핵심 기술 주제로 삼는 자바 백엔드 포트폴리오 프로젝트다.

## 2. 프로젝트 목표

이 프로젝트의 목표는 다음 네 가지를 실무형 백엔드 관점에서 보여주는 것이다.

- Spring Security 기반 인증/인가 및 역할 분리
- 상품, 장바구니, 주문, 쿠폰 중심의 커머스 핵심 CRUD
- Redis + DB 혼합 구조를 이용한 선착순 쿠폰 동시성 처리
- 관리자 주도 정산 배치 실행과 실행 이력 관리

Spring Security 공식 문서의 세션 관리, 세션 고정 보호, 동시 세션 제어, CSRF 방어 흐름을 반영해, 본 프로젝트는 Spring Security + Spring Session + Redis 기반의 stateful 세션 인증 구조를 채택한다.

Spring Batch 문서가 강조하는 job/step, 재시작 가능성, chunk 단위 처리 개념을 반영해, 정산 기능은 배치 기반 관리 기능으로 설계한다.

참고:
- [Spring Security Reference](https://docs.spring.io/spring-security/reference/servlet/getting-started.html)
- [Spring Batch Docs](https://github.com/spring-projects/spring-batch/tree/main/spring-batch-docs)

## 3. 핵심 사용자

- 구매자
  상품 조회, 장바구니 관리, 주문 생성, 쿠폰 사용, 주문 취소를 수행한다.
- 판매자
  상품 등록/수정/삭제, 재고 관리, 자신이 등록한 상품의 주문 내역 조회를 수행한다.
- 관리자
  선착순 쿠폰 정책 관리, 회원/상품/주문 조회, 정산 배치 실행 및 실행 이력 조회를 수행한다.

## 4. MVP 범위

MVP는 아래 범위까지만 포함한다.

- 회원가입, 로그인, 로그아웃, CSRF 토큰 조회
- 역할 기반 인가: 구매자 / 판매자 / 관리자
- 상품 등록, 조회, 수정, 삭제
- 상품 재고 관리
- 장바구니 담기, 수량 변경, 삭제
- 장바구니 체크아웃 및 바로 구매
- 주문 생성 및 모의 결제
- 모의 결제 완료 즉시 주문 완료 처리
- 플랫폼 공통 선착순 쿠폰 발급 및 주문 시 쿠폰 적용
- 주문 취소 및 재고 복구
- 관리자 수동 정산 배치 실행
- 정산 실행 결과 및 이력 조회

## 5. MVP 제외 범위

아래 항목은 MVP에서 제외한다.

- 외부 PG사 실결제 연동
- 배송 추적 및 송장번호 관리
- 환불/반품 프로세스
- 실시간 채팅
- 추천 시스템
- MSA 전환
- 판매자별 개별 쿠폰 정책
- 자동 스케줄 기반 정산 실행
- remember-me 자동 로그인

## 6. 핵심 기술 포인트

### 인증/인가/보안

Spring Security 기반 세션 인증과 Spring Session + Redis 저장소를 적용한다.
역할별 접근 제어와 메서드 수준 인가를 통해 구매자, 판매자, 관리자 권한을 분리하고, CSRF 방어와 세션 보안까지 함께 설계한다.

### 커머스 정합성

장바구니, 주문, 재고 차감, 주문 취소 및 재고 복구 흐름을 트랜잭션 관점에서 설계한다.
이를 통해 단순 CRUD가 아니라 상태 전이와 데이터 일관성을 보여준다.

### 쿠폰 동시성 처리

플랫폼 공통 선착순 쿠폰은 Redis를 통해 수량 차감과 중복 발급 제어를 수행하고, DB에는 최종 발급 이력을 영속화한다.
이를 통해 대규모 동시 요청 환경에서 정합성을 고려한 설계를 보여준다.

### 정산 배치

정산은 관리자가 수동 실행하는 배치 작업으로 처리한다.
배치 실행 단위, 대상 선정, 상태 변경, 실행 이력 조회를 통해 운영성 있는 백엔드 기능을 구현한다.

## 7. 기대 효과

이 프로젝트는 단순한 쇼핑몰 클론이 아니라, 직거래 도메인을 바탕으로 보안, 트랜잭션, 동시성, 배치 처리 역량을 함께 보여주는 포트폴리오를 목표로 한다.

특히 한국 백엔드 실무에서 자주 등장하는 주문/정산/운영 관리 문제를 작지만 설명 가능한 범위로 압축해 구현하는 데 의미가 있다.
