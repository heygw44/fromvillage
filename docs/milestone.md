# fromVillage Milestone and Issue Backlog

## 1. 문서 목적

본 문서는 fromVillage MVP를 GitHub 이슈 단위로 구현하기 위한 실행형 백로그다.
마일스톤별 목표만 나열하지 않고, 실제 구현 순서와 선행조건, 완료 기준, 핵심 테스트까지 함께 정의한다.

## 2. 운영 원칙

- 이슈는 하나의 책임과 하나의 검증 목표를 가진다.
- 구현 순서는 화면이나 API 노출 순서보다 선행조건과 통합 테스트 가능 시점을 우선한다.
- GitHub 이슈 번호는 생성 시점에 달라질 수 있으므로, 문서에서는 `M1-01` 같은 문서 ID를 기준으로 관리한다.
- 각 마일스톤의 마지막 이슈는 테스트 보강과 문서 정합성 점검을 포함한다.
- 인증 정책은 `docs/adr/001-auth-strategy.md`를 기준으로 고정한다.
- 주문, 쿠폰, 정산 규칙은 PRD, ERD, Architecture, API, Test Strategy 문서와 함께 검증한다.

## 3. GitHub 이슈 작성 규칙

- 제목은 `[feat]`, `[chore]`, `[test]` 등 Conventional Commit 스타일 접두사를 사용한다.
- 본문은 기능 요약, 배경 및 목적, 상세 요구사항, 완료 조건, 참고 자료를 포함한다.
- 모듈은 이슈 템플릿의 관련 모듈 필드에 기록하고, 라벨은 성격과 보안 성격 위주로 부여한다.
  예: 모듈 `auth`, 라벨 `security`, `feat`
- 문서에 적힌 선행 이슈가 닫히기 전에는 후속 이슈를 시작하지 않는다.

## 4. 마일스톤 백로그

### M1. 인증/인가 기반

- 사용자 모델, 공통 API 계약, 세션 기반 인증, 보안 정책, 인증 API를 안정적으로 구축한다.

- 완료 조건:
  - 배치가 SELLER 기준 정산 결과를 생성한다.
  - 중복 정산 방지 규칙이 동작한다.
- 핵심 테스트:
  - 배치 대상 선정 테스트
