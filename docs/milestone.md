# fromVillage Milestone

## 1. 문서 목적

본 문서는 fromVillage MVP의 구현 순서와 완료 기준을 정의한다.
문서 설계가 완료된 이후, 구현을 어떤 순서로 진행할지 단계별로 나눈다.

## 2. 마일스톤 원칙

- 보안과 인증 기반을 먼저 구축한다.
- 커머스 기본 흐름은 상품 -> 장바구니 -> 주문 순서로 확장한다.
- 쿠폰 동시성과 정산 배치는 핵심 기술 주제로 별도 마일스톤으로 분리한다.
- 각 마일스톤은 기능 구현과 테스트 기준을 함께 가진다.

## 3. 마일스톤

### M1. 프로젝트 베이스 및 인증/인가

범위:
- Spring Boot 기본 프로젝트 구성
- 사용자/역할 모델 구현
- 회원가입, 로그인, 로그아웃, CSRF 토큰 조회 API 구현
- Spring Session + Redis 기반 세션 인증 구현
- 동시 세션 1개 제한, 세션 고정 보호, 세션 쿠키 정책 구현
- 로그인 실패 횟수 제한 및 잠금 정책 구현
- Spring Security URL 보안 + 메서드 보안 설정
- 공통 응답/공통 예외 계약 구현

완료 기준:
- 인증 API가 동작한다.
- USER/SELLER/ADMIN 권한 분리가 동작한다.
- 보호 API 접근 제어 통합 테스트가 통과한다.

### M2. 상품 및 장바구니

범위:
- 상품 등록/수정/삭제/조회 API 구현
- 재고 관리 구현
- 대표 이미지 1장 처리 구조 구현
- 장바구니 담기/수정/삭제/조회 API 구현

완료 기준:
- SELLER는 본인 상품만 관리할 수 있다.
- USER는 상품 조회와 장바구니 기능을 사용할 수 있다.
- 상품 목록 검색/필터/정렬/페이지네이션이 동작한다.

### M3. 주문 및 취소

범위:
- 장바구니 체크아웃 구현
- 바로 구매 구현
- 체크아웃 시 판매자별 주문 분리 구현
- 주문 완료 시 재고 차감 구현
- 주문 취소 시 재고 복구 구현

완료 기준:
- 장바구니 주문과 바로 구매가 공통 체크아웃 흐름으로 동작한다.
- 주문 상태 전이가 정상 동작한다.
- 주문 취소 시 재고가 복구된다.

### M4. 쿠폰 정책 및 동시성 발급

범위:
- 관리자 쿠폰 정책 생성/오픈/종료 API 구현
- 사용자 쿠폰 발급/보유 조회 API 구현
- 주문 시 쿠폰 적용 구현
- Redis + DB 기반 선착순 쿠폰 발급 구현
- DB 저장 실패 시 Redis 보정 로직 구현

완료 기준:
- 동일 사용자 중복 발급이 방지된다.
- 발급 수량 초과가 발생하지 않는다.
- 주문 취소 시 쿠폰이 복구된다.
- 동시성 테스트가 통과한다.

### M5. 정산 배치 및 관리자 운영

범위:
- 관리자 회원 목록/권한 부여 API 구현
- 관리자 주문 조회 API 구현
- Spring Batch 기반 정산 배치 구현
- 정산 배치 실행 API 및 실행 이력 조회 API 구현

완료 기준:
- 정산 대상은 `COMPLETED + 미정산 주문`만 포함된다.
- 재실행 시 중복 정산이 발생하지 않는다.
- 배치 실행 이력이 저장된다.
- 배치 테스트가 통과한다.

### M6. 품질 보강 및 문서 정합성

범위:
- 테스트 전략에 정의한 핵심 테스트 보강
- OpenAPI 문서 점검
- 문서와 구현 간 용어/응답 형식/권한 규칙 정합성 점검

완료 기준:
- 핵심 회귀 테스트가 존재한다.
- API 문서와 실제 응답 구조가 일치한다.
- 문서와 구현 간 주요 규칙 불일치가 없다.
