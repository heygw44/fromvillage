# fromVillage Glossary and Role Matrix

## 1. 문서 목적

이 문서는 fromVillage 프로젝트에서 사용하는 핵심 도메인 용어와 역할별 권한 범위를 정의한다.
이후 작성되는 PRD, ERD, Architecture, API 문서는 본 문서의 용어와 권한 체계를 기준으로 한다.

## 2. 역할 정의

### USER

일반회원 역할이다.
상품 조회, 장바구니 관리, 주문 생성, 쿠폰 발급 및 사용, 본인 주문 조회와 취소가 가능하다.

### SELLER

판매자 역할이다.
기본적으로 회원은 USER로 가입하며, SELLER 권한은 ADMIN이 부여한다.
MVP에서는 USER와 SELLER를 동시에 가지지 않고, ADMIN 승인 시 USER에서 SELLER로 역할이 전환된다.
MVP에서는 SELLER를 USER 기능과 분리된 판매 전용 역할로 본다.
SELLER는 본인 상품 등록, 수정, 삭제, 재고 관리, 본인 상품이 포함된 주문 내역 조회가 가능하다.
SELLER는 본인 상품 목록을 관리 목적으로 조회할 수 있다.
SELLER의 주문 조회 범위는 고객 관점의 전체 주문이 아니라 본인 상품이 포함된 판매자 주문 단위다.

### ADMIN

플랫폼 운영 역할이다.
회원, 상품, 주문 전반을 운영 관점에서 조회하고 판매자 권한을 부여하며, 쿠폰 정책과 정산 배치를 운영한다.
ADMIN은 일반 구매자나 판매자처럼 장바구니, 주문 생성, 쿠폰 발급, 상품 등록을 수행하지 않는다.
ADMIN 계정은 공개 회원가입으로 생성하지 않고, 초기 시드 또는 운영 초기화 절차로만 생성한다.

## 3. 인증 및 인가 용어

### 회원

플랫폼에 가입한 사용자 계정이다.

### 인증

사용자의 신원을 확인하는 과정이다.
본 프로젝트는 서버 세션 기반 인증을 사용한다.

### 로그인 잠금

로그인 실패가 연속으로 누적되었을 때 일정 시간 로그인 시도를 차단하는 보호 정책이다.
본 프로젝트는 5회 연속 실패 시 10분 동안 로그인을 잠근다.

### 세션

로그인한 사용자의 인증 상태를 서버가 보관하는 저장 단위다.
본 프로젝트는 Spring Session을 통해 세션을 Redis에 저장한다.

### CSRF

세션 기반 인증에서 브라우저가 자동 전송하는 쿠키를 악용한 요청 위조를 막기 위한 보호 장치다.
본 프로젝트는 CSRF 보호를 활성화하고 `/api/v1/csrf` 엔드포인트를 통해 토큰을 발급한다.

### 인가

인증된 사용자가 어떤 기능과 자원에 접근할 수 있는지 결정하는 과정이다.

### 권한

역할에 따라 허용된 기능 범위다.
본 프로젝트는 역할 계층의 자동 상속보다 명시적 인가를 우선한다.

## 4. 상품 및 주문 용어

### 상품

SELLER가 등록한 판매 단위다.
MVP 범위의 상품 카테고리는 농산물과 수산물로 제한한다.
상품 삭제는 물리 삭제가 아니라 soft delete로 처리하며, 삭제된 상품은 신규 노출과 주문 대상에서 제외한다.

### 재고

현재 주문 가능한 상품 수량이다.
주문 완료 시 차감되고 주문 취소 시 복구된다.

### 장바구니

USER가 구매 예정 상품을 임시로 담아두는 공간이다.

### 주문

USER가 상품 또는 장바구니 정보를 기반으로 생성한 고객 관점의 주문 단위다.
한 번의 체크아웃으로 하나의 주문이 생성되며, 내부적으로는 판매자별 주문으로 분리될 수 있다.

### 판매자 주문

하나의 주문 안에서 SELLER별로 분리된 내부 처리 단위다.
정산, 판매자 주문 조회, 쿠폰 적용은 판매자 주문 기준으로 처리된다.

### 주문 완료

모의 결제가 정상 완료되어 주문이 확정된 상태다.
MVP에서는 배송 단계를 두지 않으므로 모의 결제 완료 즉시 주문 완료로 간주한다.

### 주문 취소

완료된 주문을 취소하고 재고를 복구하는 처리다.
USER는 본인 주문만 취소할 수 있다.
SELLER와 ADMIN은 MVP에서 주문 취소 권한이 없다.

## 5. 쿠폰 용어

### 쿠폰 정책

ADMIN이 생성하고 운영하는 선착순 쿠폰의 기준 정보다.
발급 수량, 정액 할인 금액, 발급 기간, 사용 조건 등을 포함한다.

### 선착순 쿠폰

정해진 수량이 모두 소진되면 더 이상 발급되지 않는 플랫폼 공통 쿠폰이다.

### 쿠폰 발급

USER가 쿠폰 정책을 기반으로 직접 발급 요청을 수행해 자신의 계정에 쿠폰을 받는 행위다.

### 쿠폰 사용

USER가 주문 생성 시 보유 쿠폰을 적용해 할인받는 행위다.

### 쿠폰 발급 이력

어떤 USER가 어떤 쿠폰을 발급받았는지 저장한 영속 데이터다.

### 중복 발급 방지

동일 USER가 같은 선착순 쿠폰을 여러 번 발급받지 못하도록 제한하는 규칙이다.

## 6. 정산 용어

### 정산

주문 완료 데이터를 기준으로 SELLER에게 정산할 금액을 계산하는 운영 처리다.

### 정산 대상

정산 배치 실행 시 계산 대상이 되는 주문 집합이다.

### 정산 배치

ADMIN이 수동으로 실행하는 정산 처리 작업이다.
MVP에서는 자동 스케줄 실행을 지원하지 않는다.

### 정산 실행 이력

정산 배치의 실행 시각, 대상 건수, 성공 여부, 실패 사유 등을 기록한 운영 데이터다.

## 8. 권한 설계 원칙

- 가입 기본 역할은 USER다.
- SELLER 권한은 ADMIN이 직접 부여하며, 승인 시 USER에서 SELLER로 역할이 전환된다.
- 이미 SELLER인 계정을 다시 SELLER로 승인하지 않는다.
- MVP에서는 USER와 SELLER를 분리된 단일 역할로 간주한다.
- 역할 계층을 강하게 의존하기보다 엔드포인트와 메서드 단위에서 명시적으로 인가한다.
- USER는 본인 리소스만 조회하거나 수정할 수 있다.
- SELLER는 본인 상품과 관련된 데이터와 본인 판매자 주문만 조회하거나 수정할 수 있다.
- SELLER 주문 조회에서 구매자 정보는 닉네임까지만 노출하고, 이메일/전화번호/주소는 노출하지 않는다.
- ADMIN은 운영 목적의 전체 조회와 관리 기능을 수행할 수 있다.
- ADMIN은 일반 구매자/판매자 기능을 겸하지 않는 운영 전용 계정으로 간주한다.
- ADMIN이 회원 정보를 조회할 때는 이메일, 닉네임, 역할, 가입일, 판매자 여부 등 기본 정보와 상태만 다루며, 비밀번호 같은 민감 인증 정보는 조회 대상에서 제외한다.
- ADMIN 계정은 공개 회원가입 대상이 아니며, 초기 시드 또는 운영 초기화 절차를 통해서만 생성한다.
- 회원 계정은 상품처럼 `deleted_at` 기반 soft delete를 바로 적용하지 않고, 계정 탈퇴/비활성화 정책을 별도 lifecycle 규칙으로 다룬다.
